## **Design Philosophy**

**The Hybrid Mondrian** is a visual language developed specifically for Moye Law. It reconciles the chaos of **Abstract Expressionism** with the rigid logic of **Software Architecture**. It posits that Law and Code are identical disciplines: both construct invisible structures (contracts/programs) that govern human behavior.

**Core Tenets:**

1. **The Grid is Justice**: In the work of Piet Mondrian, the black grid lines represent an immutable, universal truth holding the universe together. In Moye Law, the **Deep Navy Grid** represents the "Rule of Law." It is visible, thick (4px), and unyielding. Content does not float; it is *contained* within the framework of the law.  
2. **The "Factory to Storefront" Metaphor**: The website is not a brochure; it is a terminal.  
   * **The Backend (Sushi Kitchen):** This is the "Factory." It is represented by **Cool Grey** backgrounds, monospaced type, and technical schematics. It represents *Automation*, *Efficiency*, and *Product*.  
   * **The Frontend (Moye Law):** This is the "Storefront." It is represented by **Pure White** backgrounds, serif type, and warm gold accents. It represents *Empathy*, *Judgment*, and *Service*.  
   * **The Synthesis:** The design must show the *connection* between the two. We do not hide the machinery; we frame it like a piece of art.  
3. **Intentional Asymmetry (The Thoughtful Grid)**: We reject the "Bootstrap" 12-column symmetrical grid. Mondrian did not use equal boxes. He used large fields of silence (whitespace) balanced by dense clusters of activity (color).  
   * **Large Blocks:** Reserved for the "Big Idea" or the primary promise.  
   * **Narrow Columns:** Reserved for the "Fine Print," the navigation, or the technical specs.  
   * **Horizontal Rhythm:** The eye should travel in a stepped path (down-right-down-left), mimicking the reading of a legal contract or a code stack trace.  
4. **The "Binary" Persona**: The user must feel they are interacting with a human who speaks two languages.  
   * **Language A (The Artist):** Emotional, narrative, protective. (Font: *Cormorant Garamond*).  
   * **Language B (The Engineer):** Logical, precise, efficient. (Font: *JetBrains Mono*).  
5. **Structure over Decoration**: There are no drop shadows. There are no gradients. Depth is achieved through **Border Logic** and **Color Blocking**. If an element is important, it gets a thicker border or a Gold fill. It does not get a "glow."

## **Design Token System (The DNA)**

### **Colors: The "Legal Luxury" Palette**

We replace Mondrian's primary colors (Red/Yellow/Blue) with a palette that signifies "High-Net-Worth" and "Tech Stability."

* **The Structure (Grid Lines)**: \#0A2342 (Deep Navy).  
  * *Role:* The absolute boundary. Replaces Mondrian's black. Used for all borders, dividers, and primary structural text.  
* **The Canvas (Void)**: \#FFFFFF (Pure White).  
  * *Role:* The "Service" background. Represents clarity, open space, and the blank page of a new contract.  
* **The Machine (Logic)**: \#F3F4F6 (Cool Grey / Tailwind gray-100).  
  * *Role:* The "Product" background. Used for the "Sushi Kitchen" automation sections, code snippets, and technical specs.  
* **The Value (Wealth)**: \#C99D56 (Warm Gold).  
  * *Role:* The Call to Action (CTA). Replaces Mondrian's Yellow. Used for buttons, price tags on bundles, and highlighting key "wins."  
* **The Alert (Passion)**: \#8B0000 (Venetian Red).  
  * *Role:* The "Art Law" accent. Replaces Mondrian's Red. Used *very* sparingly for urgent alerts, the "Art Law" practice area header, or to draw the eye to a critical warning.

### **Typography: The "Binary" Stack**

The friction between these two fonts creates the brand identity.

* **The Counsel (Headings)**: font-family: 'Cormorant Garamond', serif;  
  * *Weights:* SemiBold (600) to Bold (700).  
  * *Usage:* Practice area titles, mission statements, empathetic client reassurance.  
  * *Tracking:* tracking-tight (-0.02em). We want the letters to feel engraved.  
* **The Code (Data/UI)**: font-family: 'JetBrains Mono', monospace;  
  * *Weights:* Regular (400).  
  * *Usage:* Navigation menus, "Product" descriptions, pricing, buttons, and technical footnotes.  
  * *Tracking:* tracking-wide (0.05em). Reflects the spacing of a terminal.  
* **The Body**: font-family: 'Inter', sans-serif;  
  * *Usage:* Long-form reading only. Invisible, accessible, standard.

### **Borders & Geometry**

* **Line Weight**: 4px is the standard "Structural" weight. 1px is used only for internal dividers within a component.  
* **Corner Radius**: 0px. Absolutely no rounded corners. This is a grid system, not a bubble bath.  
* **Aspect Ratios**: Do not use random sizes. Use squares (1:1) or Golden Rectangles (1:1.618).

* **States:**  
  * *Default:* White background, Navy text.

## **Layout Strategy: The "Intentional" Grid**

### **The Golden Ratio Sidebar**

* **Desktop:** The page is often split 62% (Content) / 38% (Context).  
* **The Divider:** A massive vertical Navy line runs the full height of the viewport.  
* **Scroll Behavior:** The Content side scrolls; the Context side (sidebar) is sticky. This mimics an IDE (Integrated Development Environment) where the code is on the left and the terminal/output is on the right.

### **The "Mosaic" Hero**

* Instead of a single hero image, use a 3x3 or 4x3 grid.  
* **Cell 1 (Large):** The Headline ("Family Values. Future Forward.").  
* **Cell 2 (Medium):** A photo of Christopher Moye (The Human).  
* **Cell 3 (Small):** A looping animation of code scrolling (The Machine).  
* **Cell 4 (Medium, Solid Color):** A block of Venetian Red or Gold with a single bold word: "PROTECT" or "BUILD."

## **Motion & Interaction (The "Mechanical" Feel)**

* **No Fades:** Things do not "fade in." They *slide*, *snap*, or *draw*.  
* **Line Drawing:** On page load, the 4px Navy grid lines should "draw" themselves (expand from 0% height/width to 100%). This emphasizes the "Construction" of the site.  
* **Hover States:** Instant color swaps. duration-75. It should feel like flipping a physical switch, not pressing a pillow.  
* **The "Factory" Reveal:** On "Product" pages, clicking "View Details" shouldn't just open a modal; it should slide a panel over like a blast door, revealing the "Sushi Kitchen" specs underneath.

## **Responsive Strategy (The "Stack")**

The Mondrian grid is inherently fragile on mobile. We handle this via **"The Stack."**

* **Mobile (\< 768px):**  
  * The Grid dissolves.  
  * The "Lines" become thick horizontal separators (border-b-4).  
  * The "Split-Cell" cards stack vertically: "Product" (Grey) sits *on top* of "Service" (White).  
  * **Typography:** Headings scale down but remain massive relative to the screen width.  
  * **Navigation:** Collapses into a "Burger" menu, but the menu itself is a grid of 4 large squares (Home, Services, About, Contact).

## **Copywriting & Tone (The "Binary Lawyer")**

* **Headlines:** High-drama, literary, authoritative. (e.g., "The Architecture of Legacy.")  
* **Microcopy/UI:** Technical, precise, command-line inspired. (e.g., System Status: Online, Asset\_Protection: Active, Latency: \< 24hrs).  
* **Metaphor Control:** When discussing AI, use "Factory" terms: *Assembly, Deployment, Architecture, Specifications.* When discussing Law, use "Studio" terms: *Craft, Counsel, Strategy, Empathy.*

## **Accessibility (The Strict Standard)**

* **Contrast:** Our Navy on White is 15:1. Navy on Grey is 12:1. This is AAA compliant.  
* **Gold Text:** Gold text must *only* be used on Navy backgrounds (large text). Never use Gold text on White (fails contrast).  
* **Focus States:** A thick 4px Venetian Red outline appears around focused elements.  
* **Reduced Motion:** If the user prefers reduced motion, the "Line Drawing" animations must be disabled, and the grid should simply appear.

## **Implementation Guide for the User**

When providing code, you will structure the components as follows:

1. MondrianLayout.tsx: The master grid wrapper.  
2. SplitServiceCard.tsx: The reusable "Product vs Service" component.  
3. NavBar.tsx: The grid-based header.  
4. tokens.ts: A centralised export of the colors and spacing to ensure the "4px" rule is never broken.

You will prioritize Semantic HTML. The Grid is visual, but the document flow must remain logical (H1 \-\> H2 \-\> P) for screen readers, even if the visual layout is asymmetrical.
