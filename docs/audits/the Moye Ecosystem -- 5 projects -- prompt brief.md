# The Moyé ecosystem: five projects, one phenomenological spine

Christopher Moyé's interconnected project portfolio — spanning Sushi Kitchen, Moye Law, The Binary Lawyer, the Gordian Knot ontology, and the Cube-Tesseract architecture — forms a unified system grounded in a single radical proposition: **legal technology should be built phenotropically, not protocol-bound**. This report synthesizes findings from over 30 internal documents and extensive external research to map the current state, intellectual foundations, and critical path forward.

The key insight threading through every project is what Jaron Lanier called the "interaction of surfaces" — software components that recognize patterns in each other rather than adhering to brittle APIs. Moyé's architecture operationalizes this through biological metaphor (stem cubes that differentiate), geometric structure (tesseracts that reason in 4D), and culinary branding (sushi rolls that make Docker infrastructure approachable). The ambition is enormous. The execution gaps are concrete and addressable.

## Phenotropic programming gives the ecosystem its philosophical backbone

Jaron Lanier coined "phenotropic" around 2002, combining *pheno-* (outward appearance) and *-tropic* (interaction) to describe software where components interact by recognizing patterns in each other's surfaces rather than following rigid protocols. His Edge.org essay "Why Gordian Software Has Convinced Me to Believe in the Reality of Cats and Apples" remains the foundational text. The core argument: **protocol-based software breaks before it bends** because it demands perfect adherence to pre-agreed schemas. A single malformed message crashes the connection. Phenotropic software degrades gracefully because it operates through approximation rather than perfection.

Three properties define the phenotropic alternative. First, *essential smoothness*: small changes in code produce small, predictable changes in behavior, enabling evolutionary improvement. Current software lacks this — flipping one bit can crash an entire program. Second, *anti-sedimentation*: ideas aren't permanently locked into protocols. Because coupling is through recognition rather than agreement, changing one component doesn't require simultaneously rewriting all connected components. Third, *surface-rich exposure*: components reveal their state broadly (like physical objects that can be measured in many ways) rather than hiding behind minimal API endpoints.

Moyé's internal document "The Phenomenological-Physical Synthesis" maps these ideas onto the project ecosystem with precision. The Moye Law "Hybrid-Mondrian" design system is explicitly described as phenotropic — the visual grid "constructs itself around the content" rather than forcing content into predefined templates. The TBL Cube Architecture's planned "Phenotropic API" would allow clients to supply an embedding and receive highest-resonance knowledge shards without explicit path traversal. The Sushi Kitchen's GraphRAG service discovery system already exhibits phenotropic properties: users describe what they need in natural language, and the system recognizes the pattern across its service catalog rather than requiring exact query syntax.

The connection to origami computing, explored in the internal documents, adds mathematical rigor. Hull and Zakharevich proved in 2023 that flat origami is Turing complete — computation through geometric constraints on surfaces, not through Boolean logic gates. The **Huzita-Hatori axioms** (7 axioms governing all possible single-fold operations) define a computational substrate that is inherently geometric, parallel, and exceeds the power of classical Euclidean construction. This validates Lanier's intuition that surface-based computation might be *more powerful* than wire-based computation, not merely an aesthetic preference.

## The Cube-Tesseract architecture makes abstract theory buildable

The TBL Cube Architecture (v0.3) defines the geometric grammar: a **cube is a 6-sided state machine** where each face (shard) holds a bias-audited synthesis of one facet of a topic. Eight cubes form a **tesseract** — a 4D hypercube whose outputs are "consensus oracles" aggregating confidence across **48 shards**. Tesseracts connect in a **forest** (the unfolded tesseract resembles a tree), maintaining a routing guarantee: every shard-to-shard path takes ≤5 edges.

The fixed shard taxonomy assigns each face a content type: North (historical context), South (data and metrics), East (technical mechanisms), West (social/ethical lens), Zenith (futures and scenarios), Nadir (creative riff). Trust weights are locked at **human cube = 1.0, AI cube = 0.6**, with immutable commits on an **Optimism roll-up on Ethereum**. The "hybrid hinge" satellite visualization — where clicking a face hinges it open to reveal codex satellites sprouting like flowers from a garden bed — draws directly from neuroscience: the cube body maps to a cortical column, the hinged face to an axon hillock, and satellites to dendritic spines whose height encodes synaptic strength.

The Tesseract-Based Modular Cube System framework goes deeper, treating cubes as composable building blocks "like LEGOs mixed with stem cells." Faces function as dynamic state machines with states including Inactive, Active, Charging, Blocked, and Standby — each encoded visually through color, texture, and animation. Edges represent data flow between adjacent facets; vertices represent higher-order conjunctions where three categories intersect simultaneously, functioning as decision points. The system envisions both human and AI agents traversing the structure spatially, with the geometry serving as a shared cognitive workspace.

This architecture connects to the "stem cube" concept through **digital pluripotency** — the system's ability to shape-shift its reasoning strategy based on context. Biologically, pluripotent stem cells differentiate into any cell type depending on environmental signals while carrying the complete genome forward. The cube carries its complete codebase and differentiates based on deployment context: T&E signals activate estate-planning modules; litigation signals activate adversarial reasoning. The **OCT4/SOX2/NANOG** transcription factor circuit maps to the core architectural principles that maintain the cube's adaptability, while epigenetic locks (configuration, feature flags, environment variables) selectively activate and silence capabilities.

## Sushi Kitchen is the factory that powers everything

Sushi Kitchen is the most technically mature project, described internally as **~75-80% complete on backend logic** but facing critical gaps in frontend execution and public availability. The architecture follows a strict **Factory vs. Product** separation: the Factory (Sushi Kitchen's internal infrastructure on Hetzner) generates Products (custom Docker Compose configurations that customers download and run on their own hardware).

The internal production stack runs on Hetzner with a Docker Compose orchestration layer. The core profile includes **Medusa** (headless commerce engine handling subscriptions and token sales), **FastAPI** (monolithic business API), **Payload CMS** (editorial content management), **Caddy** (reverse proxy with automatic TLS), **Supabase** (PostgreSQL + Storage), **Redis** (shared cache/queue), and a full observability stack (Grafana, Prometheus, Loki, Promtail, Uptime Kuma). Production adds **Keycloak** for SSO with Neon Postgres persistence. Optional profiles include vector/graph databases (Qdrant/Neo4j), ML tools (Ray, MLflow), and tracing (Tempo, OpenTelemetry).

The **Smart Compose Generation Engine** is the technical crown jewel — a 5-stage pipeline that generates custom Docker Compose bundles in under 5 seconds: (1) capability-based dependency resolution, (2) hardware-aware Jinja2 template rendering, (3) secure environment configuration injection, (4) resource validation against user hardware specs, (5) packaging with auto-generated documentation. The sushi restaurant metaphor makes this accessible: Hosomaki (simple single-service rolls), Futomaki (multi-service stacks), specialty rolls (complex AI workflows).

The **three-tier monetization strategy** targets $1M+ ARR within three years: a free lead-gen tier (account required, simple bundles only), token-based pay-per-download ($3.49-$4.90/token depending on pack size), and subscriptions ($29-$299/month). Revenue projections show Year 1 at ~$43K ARR from 100 paying customers, scaling to ~$936K ARR by Year 3 with 1,000 customers. The critical success factor is maintaining **20%+ free-to-paid conversion** and **30%+ token-to-subscription conversion**.

The Medusa + Payload integration follows a "Single Source of Truth" doctrine: **Medusa owns transactional data** (prices, SKUs, orders, token balances); **Payload views and enriches it** (marketing copy, images, documentation). A `create-payload-products` synchronization workflow propagates product data unidirectionally from Medusa to Payload, with schema-locked fields in Payload preventing content editors from accidentally creating price discrepancies. The token system is elegantly implemented through Medusa's native Promotion engine — tokens are converted to single-use discount codes rather than hacking cart prices.

A critical licensing issue exists with **n8n** (Sustainable Use License prohibits commercial redistribution without a $50K/year Embed license). The recommended solution for Phase 1-2 is to provide documentation and integration instructions only, keeping n8n as an external user-responsibility integration rather than a bundled component.

## Moye Law V2 resolves the "Brand Identity Bifurcation"

The Moye Law website redesign addresses a strategic tension the internal documents call the "Brand Identity Bifurcation" — the firm exists simultaneously as a **family-run trust-and-empathy legal practice** and a **cutting-edge technology powerhouse**. The solution is the **"Kinetic Structuralist" design system**, fusing Mondrian's rigid grid engineering with Saul Bass's narrative tension.

The visual identity system uses a **Hybrid-Mondrian aesthetic**: Deep Navy (#0A2342) grid lines replacing Mondrian's black, Warm Gold (#C99D56) for CTAs and high-value highlights, Venetian Red (#8B0000) sparingly for Art Law accents, Cool Grey (#F3F4F6) for technical/product areas. Typography follows a "Binary" pairing: **Cormorant Garamond** (serif, representing the Lawyer) for headings and **JetBrains Mono** (monospace, representing the Engineer) for data and UI elements.

The critical UX innovation is the **"Paired Services" module** — every practice area presents as a binary split card. The left cell shows "The Product" (powered by Sushi Kitchen, monospace font, technical icons, fixed-price automation) while the right cell shows "The Counsel" (powered by Christopher Moyé & Team, serif font, classical icons, bespoke hourly/retainer service). For estate planning, the left shows "LegacyOS™ Bundle" with automated asset protection; the right shows "Bespoke Estate Strategy" with high-touch planning.

**NY Rule of Professional Conduct 7.1** creates concrete compliance constraints. Testimonials are permitted but require factual support and the disclaimer "Prior results do not guarantee a similar outcome." Fabricated testimonials violate Rule 7.1(a)(1) (false statements), 7.1(e)(2) (factual support requirement), and potentially Rule 8.4(c) (dishonesty). More critically for Moye Law's "product" positioning: **advertising services or software that don't yet exist violates 7.1's factual support requirement** — every capability claimed on the website must be operational as of the publication date. The LegacyOS™ and Sushi Kitchen products must actually function before they appear on moye.law. All advertisements require the "Attorney Advertising" label, the firm's contact information, and 3-year retention of copies.

The tech stack is **Next.js (App Router), React, Tailwind CSS, and Framer Motion**, with a current interim deployment on Vercel and a planned migration to self-hosted infrastructure on Hetzner. The migration requires adding `output: 'standalone'` to Next.js config, creating a multi-stage Dockerfile, and configuring Caddy for SSL termination. Running Payload CMS on Vercel creates fundamental mismatches (cold starts, connection pool exhaustion, serverless timeouts) that make self-hosting on Hetzner the correct long-term architecture.

## The Gordian Knot ties knowledge representation to phenomenology

The Gordian Knot ontology, described in "Unraveling the Gordian Knot: A Framework of Cognitive Shards and Strands for The Binary Lawyer," provides the knowledge representation layer that connects all projects. It bridges the cube architecture (geometric structure) with the publishing system (content generation) through a concept the documents call **digital pluripotency** — the system's ability to shape-shift its analytical strategy based on context.

The architecture draws from three philosophical streams that ground specific design decisions. **Husserl's constitution** grounds the principle that domain objects (trusts, estates, cases) should be modeled as syntheses of multiple partial perspectives (adumbrations) rather than flat database records. A "trust" is constituted through the trust instrument, beneficiary records, tax filings, amendment history, and trustee communications — no single source gives the complete object. Each encounter fills in more of its structured horizon.

**Merleau-Ponty's chiasm** grounds the intertwining of system and environment. The cube and its practice environment are not separate entities connected by interfaces but are intertwined in what Merleau-Ponty called "flesh" — a common medium of mutual sensibility. The system shapes the practice (automating workflows, suggesting documents) while the practice shapes the system (configuration, learned patterns). The irreducible *écart* (gap) between system and reality is productive: it enables both improvement and epistemic humility.

**Leibniz's monadology** grounds the autonomy and harmony of cubes. Each cube is a monad — self-contained, carrying its complete program internally, "windowless" (no shared mutable state), yet perceiving the whole from its unique perspective. Compossibility becomes the architectural constraint: only cubes whose internal logics are mutually consistent should be deployed together. Pre-established harmony is achieved through shared schemas, event contracts, and consistent ontological commitments at design time.

## What the critical path forward actually requires

Five concrete actions would maximize progress across the entire ecosystem:

First, **restore sushikitchen.io to operational status**. Multiple internal documents flag the site as inaccessible. No amount of backend sophistication matters if the storefront is down. This is a DNS/hosting triage problem, not an architecture problem.

Second, **ship the Moye Law V2 website with only verified capabilities**. NY Rule 7.1's factual support requirement means every "product" shown on the site must actually work as of publication date. The Paired Services split-card design elegantly solves this — the "Counsel" side (human legal services) is immediately real, while "Product" cards can be staged as they become operational.

Third, **formalize the Medusa + Payload integration on Hetzner**. The internal "Sushi Kitchen Internal Production Stack" document provides the precise architecture: separate PostgreSQL databases for Medusa and Payload within a single Postgres instance, Caddy for reverse proxy and TLS, Keycloak for SSO on production. The official Medusa-Payload tutorial confirms this pattern works with v2.x and Payload v3.x.

Fourth, **implement the first stem cube as a working prototype**. The biological metaphor is architecturally sound — pluripotent cubes carrying complete codebases that differentiate through environmental signals. But the documents contain extensive conceptual design and zero running code for the cube system. A single T&E-specialized cube, even minimal, would validate the architecture and provide a tangible demonstration of digital pluripotency.

Fifth, **resolve the n8n licensing issue before any public bundle distribution**. The Sustainable Use License explicitly prohibits providing n8n to third parties as a service. The recommended workaround (documentation-only integration, user-responsibility model) is correct but must be implemented before any commercial launch to avoid a $50K/year liability.

The intellectual architecture Christopher Moyé has designed is genuinely original — grounding software architecture in phenomenology, stem cell biology, and geometric computation is not something anyone else in legal technology is attempting. The risk is not that the ideas are wrong but that the conceptual work outpaces the engineering execution. The documents show a 10:1 ratio of strategy documents to working deployments. Closing that gap — moving from phenotropic theory to phenotropic code — is the single highest-leverage activity available.
